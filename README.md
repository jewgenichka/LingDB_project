# LingDB
База лингвистических задач.

[Ссылка на телеграм-бота](https://web.telegram.org/a/#8745809932)

## Описание проекта
Телеграм-бот для добавления, хранения и поиска олимпиадных лингвистических задач.

Пользователи могут добавлять задачи с метаданными (автор, язык, олимпиада, год). После отправки задачи проходят модерацию, сохраняются в базе данных и доступны для поиска.

Бот поддерживает поиск по метаданным и позволяет хранить задачи структурировано.

## Возможности

- добавление задач пользователем
- модерация добавленных пользователем задач:
	- одобрение
	- редактирование
- хранение одобренных задач
- поиск по:
	- автору
	- тегам
	- языку
	- олимпиаде
	- годам
- хранение Telegram file_id
- навигация через inline-кнопки

## Техническая база

- ***Python*** - основная логика проекта
- ***aiogram*** — работа с Telegram Bot API  
- ***SQLite*** — локальное хранение данных (встроен в python) 
- ***APScheduler*** — планировщик задач  
- ***python-decouple*** — управление конфигурацией через .env

Полный список можно посмотреть в [requirements.txt](https://github.com/jewgenichka/LingDB_project/blob/master/requirements.txt).

## Структура репозитория
Бот запускается сейчас и без консоли, но если необходимо включить его на локальном компьютере, то запускать необходимо файл aiogram_run.py. Сам бот и его данные лежат в create_bot.py. Все функции бота, необходимые админам, лежат в admin.py и импортируются в aiogram_run.py. База данных - olympiad_tasks.db, в ней хранятся все задачи, и с ней связывается db.py. До создания бота, еще на этапе разработки db.py, все тесты делались в test.py, сейчас этот файл весьма бесполезен, хотя с его помощью можно загружать задачи вообще без ответа и оставить хихихихи пасхалку. 

## Логика работы 
### Взаимодействие с пользователем
Происходит через команды и кнопки. Добавление задачи происходит пошагово: бот запрашивает данные, пользователь вводит (название, язык, текст задачи и так далее).

### Обработка данных 
Для обработки данных задачи используется логика пошагового ввода. Она позволяет запоминать, на каком этапе добавления задачи находится пользователь и какие данные он сейчас вводит.

### Хранение данных
Все данные сохраняются в SQLite базе данных.

Чтобы не хранить файлы на сервере, используется Telegram file_id.

### Модерация
После добавления задача должна пройти модерацию, прежде чем стать доступной пользователям. Администраторы могут одобрить или отклонить задачу.

### Поиск
Пользователь может искать задачи по параметрам:
- автор
- язык
- олимпиада
- год
- теги
